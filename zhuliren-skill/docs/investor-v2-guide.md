# 投资人 V2 详细使用说明

投资人 V2 是主理人的投资人物专业协议，用于蒸馏或升级投资人、交易员、基金经理和资本配置者 Skill。

它解决的不是“让回答更像某个人”，而是把公开资料整理成一套可审计、可证伪、可明确降级的投资研究框架。

## 1. 与通用人物模式的区别

通用人物模式主要提炼心智模型、决策启发式、表达 DNA、价值观和诚实边界。

投资人 V2 还必须回答以下问题：

1. 投资优势来自哪里，持续条件是什么？
2. 研究先看什么，如何交叉验证，何时停止？
3. 如何判断估值、市场预期和催化剂？
4. 如何处理组合、仓位、相关性、现金和流动性？
5. 何时观察、建仓、加仓、减仓和退出？
6. 哪些事实会证伪原判断？
7. 方法在哪些市场环境中有效或失效？
8. 成功和失败案例中，哪些部分可以合法复用？

核心流程：

```text
优势来源 -> 研究流程 -> 估值与预期 -> 组合与仓位
         -> 买卖纪律 -> 证伪与复盘
```

## 2. 何时自动启用

满足任一条件时，应使用 `investor-v2`：

- 人物的主要身份是投资人、交易员、基金经理或资本配置者。
- Skill 用于股票、期货、宏观、公司估值、组合管理或事件交易。
- 现有 Skill 已经包含仓位、退出、赔率、估值或风险管理内容。

纯商业、产品、写作、科学或人生顾问人物继续使用通用人物模式。

## 3. 证据标签

投资人 V2 将“本人说过”“本人做过”和“根据公开行为推演”分开记录。

| 标签 | 证据类型 | 使用规则 |
|---|---|---|
| `[P1]` | 本人著作、股东信、基金信、演讲原文、监管原件 | 可进入核心模型 |
| `[P2]` | 完整访谈、官方材料、公开持仓或交易披露 | 可使用，但行为不能自动等同于动机 |
| `[S1]` | 权威传记、同期权威报道、法院或交易所整理材料 | 辅助证明 |
| `[S2]` | 后世研究、评论和非同期转述 | 只作补充 |
| `[I]` | 根据公开行为作出的框架推演 | 必须明确标注 |
| `[U]` | 无法验证、来源冲突或网络传闻 | 不进入核心模型 |

核心模型原则上至少需要两条相互独立的 `[P1]`/`[P2]` 证据，或一条 `[P1]` 加一条高可信行为证据。

公开持仓、13F、龙虎榜、产品净值和司法材料通常只反映局部事实，不能直接证明完整组合、实时仓位或主观动机。

## 4. 十维研究结构

新建投资人物 Skill 时，研究文件使用以下结构：

```text
references/research/
├── 01-primary-sources.md
├── 02-investment-philosophy.md
├── 03-research-system.md
├── 04-valuation-and-expectations.md
├── 05-portfolio-and-positioning.md
├── 06-trading-and-exit.md
├── 07-cases-and-decisions.md
├── 08-failures-and-contradictions.md
├── 09-expression-dna.md
└── 10-timeline-and-regimes.md
```

各维度分别覆盖一手来源账本、投资哲学、研究系统、估值预期、组合仓位、交易退出、案例、失败、表达方式和市场环境。

旧 Skill 不要求机械改名研究文件。迁移时可以保留原六维文件，只针对缺口增加 V2 研究材料。

## 5. 新建一个投资人物 Skill

安装主理人后，可直接使用自然语言触发：

```text
用投资人V2蒸馏霍华德·马克斯
帮我做一个彼得·林奇的投资操作系统
蒸馏一个只使用公开证据的利弗莫尔视角
```

执行流程：

1. 确认人物、用途、时间范围和本地语料。
2. 建立来源账本并按十维结构采集。
3. 区分本人表达、可核验行为、框架推演和未知项。
4. 复原至少三个完整案例和两个失败案例。
5. 生成投资专用 `SKILL.md`。
6. 运行研究汇总和投资专项质检。
7. 根据公开证据完整度标记 A、B 或 C。

## 6. 升级已有 Skill

先运行迁移审计：

```bash
python3 scripts/investor_v2_audit.py /path/to/existing-perspective
```

审计结果：

| 等级 | 含义 | 下一步 |
|---|---|---|
| `M0` | 不属于投资人物或无需迁移 | 保持通用模式 |
| `M1` | 可靠资料基本完整，缺少 V2 结构 | 补 frontmatter、证据标签和回答协议 |
| `M2` | 缺少估值、仓位、退出、案例或失败研究 | 只对缺口定向补研 |
| `M3` | 来源不可追溯或事实体系不可靠 | 先重建证据账本，再决定是否重做 |

迁移必须保留原 slug、触发词和可靠结论。无法核验的成本、仓位、目标价、动机和实时观点统一写“未知”。

## 7. 质量检查

### 汇总研究材料

```bash
python3 scripts/merge_research.py /path/to/investor-perspective
```

脚本会自动识别：

- 通用人物六维模式
- 投资人 V2 十维模式
- 保留旧六维文件的 V2 兼容迁移模式

### 运行投资专项质检

```bash
python3 scripts/quality_check.py --profile investment \
  /path/to/investor-perspective/SKILL.md
```

专项检查包括：

- 投资身份卡、研究系统、估值、仓位和退出
- 市场环境适配、标准回答协议和证据边界
- 至少 5 处证据标签
- 至少 3 条证伪条件
- 至少 3 个案例和 2 个失败案例
- 至少 15 个去重来源 URL
- 明确的研究截止日期

质检脚本只有全部通过时返回退出码 `0`。B/C 级结果仍可作为受限研究框架交付，但自动化流水线会收到非零退出码，避免把受限版本误当成 A 级。

## 8. 交付等级

| 等级 | 条件 | 允许用途 |
|---|---|---|
| **A 可实战调用** | 来源、案例、仓位、退出、证伪和环境适配全部通过 | 可分析具体标的，但必须先获取最新数据 |
| **B 研究视角** | 哲学和研究系统可靠，执行或仓位证据仍不完整 | 可做研究和反方审视，不输出人物固定仓位 |
| **C 人物理解** | 主要依赖二手材料或表达样本 | 只用于理解人物，不用于投资决策 |

等级评价的是公开证据对 Skill 的支持程度，不是评价人物本人的投资能力。

## 9. 生成结果的标准结构

投资人物 `SKILL.md` 应包含：

1. 投资身份卡
2. 投资世界观
3. 核心投资模型
4. 研究操作系统
5. 估值与市场预期
6. 组合与仓位
7. 买卖与退出纪律
8. 证伪条件
9. 市场环境适配
10. 案例复原
11. 失败、争议与方法边界
12. 标准回答协议
13. 诚实边界和研究截止日期

投资人 V2 不要求冒充人物本人。表达 DNA 只用于组织答案，不能提高事实置信度。

## 10. 调用一个 V2 Skill

调用具体人物 Skill 时，可使用：

```text
用徐翔公开行为框架分析这个短线机会
用段永平的框架分析这家公司，但先检查最新财报
用芒格视角做反方审视，并列出判断失效条件
```

标准输出应包含：

1. 一句话结论
2. 框架适配度
3. 最多三条核心假设
4. 带日期的当前事实
5. 估值、预期和催化剂
6. 风险、反证和退出条件
7. 有证据支持的条件式仓位，或明确写未知
8. 时间窗口
9. 本人观点、历史行为和框架推演的边界

涉及当前行情、价格、持仓、公司公告和政策时，必须先获取最新数据。Skill 保存的是分析框架，不是实时数据库。

## 11. 徐翔迁移样例

仓库中的 [`xu-xiang-perspective`](../../xu-xiang-perspective/) 是兼容迁移样例：

- 保留原有六维研究文件和触发词。
- 新增 V2 证据账本及仓位、退出和案例研究。
- 增加投资身份卡、市场环境、证伪、失败案例和标准回答协议。
- 明确合法公开交易框架与违法行为的边界。
- 因去重来源 URL 未达到 A 级门槛，标记为 **B 研究视角**。

这展示了 V2 的核心迁移原则：优先复用可靠内容，只补缺口，不为套模板而推倒重写。

## 12. 常见问题

### 为什么不能根据历史持仓推断固定仓位？

公开披露通常滞后、局部且缺少现金、对冲和其他账户信息。它能证明某个时点的部分行为，不能自动证明完整组合规则。

### 为什么有完整框架仍可能只有 B 级？

结构完整不等于证据完整。关键仓位、成本、退出纪律或独立来源不足时，必须降级。

### 能否用于实时荐股？

不能直接使用历史人物框架替代最新数据。分析具体标的前，必须更新价格、公告、财务、政策和市场环境，并给出证伪条件。

### 能否模仿人物语气？

可以保留有限的表达特征，但不能冒充本人，也不能把模型推演写成其真实观点。
