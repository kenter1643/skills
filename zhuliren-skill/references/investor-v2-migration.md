# 投资人 V2：旧 Skill 迁移协议

本协议用于升级已经蒸馏的人物 Skill。默认增量迁移，不推倒重写，不覆盖已有可靠内容。

## 1. 先判断是否应迁移

满足任一条件即进入投资人 V2 审计：

- 人物的主要身份是投资人、交易员、基金经理或资本配置者。
- Skill 用于股票、期货、宏观、组合或公司估值。
- 现有 Skill 已包含仓位、退出、赔率、估值或风险管理内容。

纯商业、写作、产品或人生顾问人物不迁移。

## 2. 审计而非重写

先运行：

```bash
python3 scripts/investor_v2_audit.py <旧Skill目录>
```

审计把旧内容映射到 V2 字段，并输出：

- 已满足：直接保留，不重复研究。
- 可迁移：旧内容可靠，但需要移动或增加证据标签。
- 待补研究：没有足够证据，必须补来源。
- 未知：公开资料无法确认，在新 Skill 中明确写未知。

## 3. 旧六文件到新十维的映射

| 旧文件 | 可迁移到 |
|---|---|
| `01-writings.md` | 01一手来源、02投资哲学、03研究系统、04估值 |
| `02-conversations.md` | 01一手来源、02投资哲学、03研究系统、06交易退出 |
| `03-expression-dna.md` | 09表达DNA |
| `04-external-views.md` | 08失败争议、10环境适配 |
| `05-decisions.md` | 05组合仓位、06交易退出、07案例、08失败 |
| `06-timeline.md` | 10时间线与市场环境 |

旧文件可以继续保留。V2 不要求机械改名；只有新增研究较多时才创建十维文件。

## 4. 四种迁移等级

### M0：无需迁移

不是投资人物，或现有用途不涉及投资决策。

### M1：结构升级

旧资料已经覆盖关键内容，只需：

- frontmatter 增加 `profile: investor-v2`
- 增加投资身份卡、市场环境适配和标准回答协议
- 为重要结论补证据标签

### M2：增量补研

已有哲学框架，但缺少以下任一项：

- 组合与仓位
- 估值与预期
- 买卖与退出
- 三个完整案例
- 两个失败案例
- 明确证伪条件

只针对缺口补研，不重新采集已经可靠的维度。

### M3：重新蒸馏

出现以下情况才重做：

- 核心内容主要来自二手转述或网络传奇。
- 人物原话、行为推断和传闻混在一起，无法拆分。
- 没有可追溯来源。
- 现有框架与新发现的一手资料发生根本冲突。

## 5. 迁移执行顺序

1. 备份或确认 Git 工作区状态。
2. 运行旧 Skill 通用质检和 V2 审计。
3. 生成缺口清单，不直接改结论。
4. 复用旧研究文件，按缺口定向补研。
5. 给核心结论添加证据等级。
6. 增加 V2 必需 section。
7. 运行 `quality_check.py --profile investment`。
8. 用一个历史案例和一个当前问题做前向测试。
9. 只有新版本更可靠时才替换旧版。

## 6. 兼容规则

- 保留旧触发词和 slug，避免安装链接失效。
- 不因模板升级改变人物既有结论，除非新证据支持。
- 无证据的具体仓位、成本和目标价统一写“未知”。
- B/C 级 Skill 不得通过添加格式字段伪装成 A 级。
- 迁移记录应说明新增内容、修正内容和仍然未知的内容。
