# API 接口文档生成器

根据 SQL 建表语句和 Excel 需求文档自动生成 Markdown 格式 API 接口文档。

## 环境

- **Python**：3.8+
- **依赖**：`pip install openpyxl pypinyin translators`

## 输入文件

| 文件 | 要求 |
|---|---|
| SQL | 包含 `CREATE TABLE`，字段建议带 COMMENT |
| Excel | 必须包含名称含"字段说明"的 sheet |

### Excel 列要求（关键字匹配）

| 列名 | 说明 |
|---|---|
| 字段名称 | 用于匹配 SQL 注释 |
| 是否必填 | 影响保存接口入参 |
| 是否列表字段 | 决定是否出现在列表出参 |
| 列表顺序 | 数字决定列表字段顺序 |
| 查询方式 | 模糊=模糊+高级；高级=仅高级；空=不加入 |
| 是否枚举 | 决定是否生成下拉接口 |
| 枚举内容 | 追加到字段备注 |

## 使用

```bash
python main.py "<sql文件>" "<excel文件>" --output "<输出目录>"
```

输出文件：`output/业务名称.md`

## 输出接口

单据列表、详情、保存、删除、撤回、审核、转交、导出、校验参数、下拉接口。

## 核心规则

- **表名提取**：`{模块}_bill_{编号}_{后缀}` → `/模块/bill/编号`
- **字段匹配**：Excel 名是 SQL 注释的前缀
- **SQL 合并**：只合并 head/detail 表，info 表用于生成 infoList
- **列表出参**：String 类型自动分配导出排序；额外带出 `projectId`（在 fxmbh 前）、`fsszzid`（在 fsszz 前）、`fztmcid`（在 fztmc 前）、`contractId`（在合同字段前）
- **主表字段**：包含 `id`（ID）和 `headId`（主单id）
- **infoList**：备注使用表 COMMENT，中文括号自动转英文
- **明细区域**：Excel 第一列以"明细"开头的合并单元格，其下方字段不参与主信息输出
- **枚举值**：换行符自动转为"、"分隔

## 常见问题

| 问题 | 原因 |
|---|---|
| 文件不存在 | 路径错误或未加引号 |
| 接口路径不对 | SQL 表名格式不符合 `{模块}_bill_{编号}_{后缀}` |
| 找不到 sheet | Excel 中无"字段说明" |
| 字段缺失 | SQL 注释与 Excel 中文名差异太大 |
| 必填/枚举未生效 | 表头列名无法识别 |

生成后建议人工复核接口路径、字段完整性和必填标识。
